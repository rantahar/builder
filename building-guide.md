# AI Design Guide — How to Build Valid Designs

This is a reference for the AI design agent. Building a design works like real-world assembly: pick a piece, orient it, attach it to a face of an existing piece. The system computes positions automatically — no coordinate calculation needed.

## Two Formats

- **Build steps JSON** — the source format. This is what you write. An ordered list of assembly steps.
- **Design JSON** — the compiled output. Generated by `core/builder.py` from build steps. Consumed by the validator, renderer, and instruction generator.

You author build steps; the system compiles them into a design.

## Building Process

Each design starts with one piece placed at the origin (`place`). Every subsequent piece is added by declaring how it attaches to an existing piece (`attach`). Additional connections between existing pieces use `connect`. Three step types:

```json
{
  "meta": { "name": "My chair", "library": "wood_basic" },
  "steps": [
    { "action": "place", "piece": "fl_leg", "type": "lumber_2x2",
      "rotation": [90, 0, 0], "length": 430 },

    { "action": "attach", "piece": "left_rail", "type": "lumber_2x2",
      "face": "-z", "to": "fl_leg", "to_face": "+z",
      "length": 310, "offset": [0, 385], "fastener": "4x70" },

    { "action": "connect", "piece": "seat", "face": "-y",
      "to": "right_rail", "to_face": "+y", "fastener": "4x50" }
  ]
}
```

The builder computes positions, adds bidirectional connections, and produces the design JSON. Schema: `core/schema/buildsteps.schema.json`.

### Step types

| Action | Purpose | Required fields |
|--------|---------|-----------------|
| `place` | First piece at origin | `piece`, `type` |
| `attach` | New piece attached to existing | `piece`, `type`, `face`, `to`, `to_face` |
| `connect` | Secondary connection | `piece`, `face`, `to`, `to_face` |

Optional on all: `rotation`, `length`, `width`, `fastener`. `offset` is optional on `attach` (default `[0, 0]`).

## Piece Orientation

Pieces have base dimensions `(width, height, length)` mapping to `(x, y, z)`. Rotation swaps these:

| Rotation | Swap | Use case |
|----------|------|----------|
| `[90, 0, 0]` | height ↔ length | Vertical post (length becomes height) |
| `[0, 90, 0]` | width ↔ length | Horizontal rail along x-axis |
| `[0, 0, 0]` | (none) | Horizontal rail along z-axis, flat panel |
| `[0, 0, 90]` | width ↔ height | Rotate cross-section 90° |

After rotation, the result is the **effective dimensions** `(eff_w, eff_h, eff_l)` — the actual (x, y, z) extent of the piece in world space.

**Example — vertical post:** `lumber_2x2` (45×45×2400) with `length_override=430` and `rotation=[90,0,0]`: height and length swap → effective `(45, 430, 45)`. A 45×45mm post standing 430mm tall.

**Example — horizontal rail along x:** `lumber_2x2` with `length_override=600` and `rotation=[0,90,0]`: width and length swap → effective `(600, 45, 45)`. A rail spanning 600mm along x.

## Face Labels

Face labels are world-axis aligned and always refer to the same direction:

| Label | Direction | Surface |
|-------|-----------|---------|
| `+y` | up | Top of the piece |
| `-y` | down | Bottom of the piece |
| `+x` | right | Right side |
| `-x` | left | Left side |
| `+z` | back | Back side |
| `-z` | front | Front side |

**Critical — available faces per piece type:**

| Piece type | Available faces | Notes |
|-----------|----------------|-------|
| `lumber_2x2`, `lumber_2x4`, `lumber_4x4` | All 6 | Most versatile — use for structural frames |
| `plank_18x95`, `plank_18x145` | `+y`, `-y` only | `screw_entry` only (cannot receive screws) |
| `plywood_18mm` | `+y`, `-y` only | Can both send and receive screws, but only on top/bottom |

**Limitation:** Planks and plywood only have `+y`/`-y` faces regardless of rotation. Even when rotated to stand vertical, only the world-top and world-bottom edges have attachments. For vertical panels or connections requiring side faces, use lumber pieces for the structural frame and connect plywood/planks only via their `+y`/`-y` faces.

## Offset — Positioning on the Contact Plane

When attaching a piece to a face, the `offset=(u, v)` parameter controls where on the face the new piece sits. The offset is measured from the **existing piece's face min corner** along the two non-normal axes.

For a `+y` or `-y` face, the contact plane axes are `(x, z)`:
- `offset=(0, 0)` → new piece's corner aligns with existing piece's (x, z) corner
- `offset=(10, 50)` → shifted 10mm along x, 50mm along z

For a `+x` or `-x` face, the contact plane axes are `(y, z)`.
For a `+z` or `-z` face, the contact plane axes are `(x, y)`.

**Example — rail flush with leg top:**
A vertical leg (effective height 430mm) has its `+z` face. The contact plane axes are `(x, y)`. To position a rail at the top of the leg: `offset=(0, 385)` shifts the rail 385mm up on the y-axis, so the rail (45mm tall) spans y=385 to y=430 — flush with the leg top.

## Connection Rules

The validator enforces these rules on every connection:

1. **Face exists** on the piece type's library definition
2. **Compatible types**: `screw_entry` ↔ `screw_receiver`. Two `screw_entry` faces cannot connect directly (planks cannot attach to planks)
3. **Bidirectional**: the builder handles this automatically via `add_piece()` and `add_connection()`
4. **Coplanar**: connected faces are at the same world coordinate — the builder computes this automatically
5. **Overlapping**: connected faces share surface area on the contact plane — choose offsets that ensure overlap

**Fastener format**: `"NxL"` — N=diameter mm, L=length mm. Example: `"4x70"` = 4mm × 70mm screw.

**Screw collision**: Screws from opposite faces of a receiver piece must not overlap inside. Combined penetration = `(screw_length - entry_thickness)` from each side. Must be less than the receiver's dimension on that axis. Use shorter screws or connect from one side only when needed.

## Build Order

The pieces array in the design JSON is the build log. Each `add_piece()` call appends one step. Rules:

- First piece: placed at origin, no attachment
- Every subsequent piece: must attach to an already-placed piece
- Order should be physically realizable — earlier pieces don't require later pieces to be stable
- Think IKEA: build flat subassemblies first, then join them

## Piece Library Quick Reference (wood_basic)

| Piece ID | Cross-section | Variable dims | Faces | Category |
|----------|--------------|--------------|-------|----------|
| `lumber_2x2` | 45×45mm | length | All 6 | post |
| `lumber_2x4` | 45×95mm | length | All 6 | post |
| `lumber_4x4` | 95×95mm | length | All 6 | post |
| `plank_18x95` | 95×18mm | length | +y, -y | plank |
| `plank_18x145` | 145×18mm | length | +y, -y | plank |
| `plywood_18mm` | any×18mm | length + width | +y, -y | panel |

**Default orientation**: width along x, height along y, length along z. For `lumber_2x4`: width=45 (x), height=95 (y), length along z.
